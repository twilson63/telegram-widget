Telegram Widget — Start/Stop Feedback

LFG PRD

Definition

Bridge transition feedback means the config screen clearly shows that a start or stop request is in progress, with a visible animated pending status and disabled controls that prevent duplicate/conflicting start/stop clicks until the operation resolves. It does not change the Telegram bridge protocol, token handling, command approval policy, or background process behavior.

Goals

  • Show an animated pending state while starting the bridge.
  • Disable Start during startup, and prevent conflicting Stop clicks during startup.
  • Show the same animated pending state while stopping the bridge.
  • Disable Stop during shutdown, and prevent conflicting Start clicks during shutdown.
  • Return controls to the correct enabled/disabled state when running, stopped, or errored.

Non-goals / constraints

  • No changes to bridge daemon, WebSocket protocol, or command strings.
  • No secrets or bot tokens in logs, docs, or command line.
  • Keep the widget as a single static HTML file; no new dependencies.
  • Keep approvals visible; do not bypass HyperDesk command.run approvals.

Plan and acceptance criteria

  1. Add reusable pending UI styling.
    Animated spinner appears inside #configStatus whenever it has class pending. CSS diff inspection.
    Animation is lightweight CSS only and does not require extra assets/scripts. CSS diff inspection.
  2. Add centralized control state helpers.
    Start/Stop labels and disabled states are derived from status/operation instead of being scattered across handlers. JS diff inspection.
    Running enables Stop only; stopped/error enables Start only; starting/stopping disables both. JS diff inspection and syntax check.
  3. Wire start and stop operations.
    Start immediately shows animated pending text and disables controls before command.run. JS diff inspection.
    Stop immediately shows animated pending text and disables controls before command.run. JS diff inspection.
    Failure paths clear the operation and restore an actionable control state. JS diff inspection and syntax check.

Validation

  • Compile the inline widget script with Node syntax checking by extracting it from ui/index.html.
  • Optionally run existing bridge tests from scripts/ if dependencies are available.
